  ArkhamHorror 本地服务器 使用教程
  生成日期: 2026-03-27
================================================================

【一、项目简介】

  你搭建了一个本地局域网版的 ArkhamHorror 游戏服务器，包含以下功能：
  - ArkhamHorror 卡牌游戏主程序
  - Arkham.build 组牌器（本地版）
  - 背景音乐播放（实时读取 music 文件夹）
  - 多人语音聊天（支持切换输入/输出设备）
  - 网页小聊天室

  局域网内的其他设备（手机、平板、电脑）都可以通过浏览器访问。


【二、文件结构】

  D:\ArkhamHorror\                  ← 主目录
  ├── start-all.ps1                 ← 一键启动脚本
  ├── update-ip.ps1                 ← IP 变更时用的脚本
  ├── https-proxy.js                ← HTTPS 代理（端口 3443 + 3445）
  ├── voice-server.js               ← 语音聊天服务（端口 3001）
  ├── tools.html                    ← 音乐/语音/聊天室功能页面（主版本）
  ├── tools-inject.js               ← 注入 tools.html 的脚本
  ├── server.pem / server-key.pem   ← SSL 证书
  ├── docker-compose.yml            ← Docker 容器配置
  ├── frontend\                     ← ArkhamHorror 前端源码
  │   ├── index.html                ← 前端入口（已注入 tools-inject.js）
  │   ├── vite.config.js            ← Vite 开发服务器配置
  │   ├── public\music\             ← 音乐文件夹（放 mp3 进去自动识别）
  │   ├── public\tools.html         ← tools.html 副本（启动时自动同步）
  │   └── public\tools-inject.js    ← tools-inject.js 副本（启动时自动同步）
  │
  D:\arkham-deckbuilder\            ← 组牌器目录
  ├── frontend\                     ← 组牌器前端
  └── local-share-api.cjs           ← 本地分享 API


【三、日常使用：开机一键启动】

  步骤：
  1. 开机，等 Docker Desktop 自动启动（或手动打开它）
第一次启动要先运行update-ip.ps1更新ip和颁发证书，以后如果ip变换也要运行
  2. 右键 D:\ArkhamHorror\start-all.ps1 → "使用 PowerShell 运行"
  3. 等待脚本显示"全部完成！"
  4. 打开浏览器访问：
     - 游戏主页:    https://你的ipv4:3443
     - 组牌器:      https://你的ipv4:3445

  就这么简单！一个脚本搞定所有事情。

  注意：如果浏览器提示"不安全"，点"高级"→"继续前往"即可。


【四、IP 地址变了怎么办】

  如果你换了 WiFi 或者路由器重新分配了 IP，需要更新：

  步骤：
  1. 先查看新 IP：打开 PowerShell，输入 ipconfig，找到 IPv4 地址
  2. 右键 D:\ArkhamHorror\update-ip.ps1 → "使用 PowerShell 运行"
  3. 输入新的 IP 地址，回车
  4. 等待脚本显示"全部完成！"
  5. 用新 IP 访问即可

  update-ip.ps1 会自动完成以下所有事情：
  - 停止旧服务
  - 生成新 SSL 证书
  - 更新所有配置文件
  - 修复 Vite 配置
  - 同步 tools 文件
  - 重新编译组牌器前端
  - 重启 Docker 容器
  - 重启所有服务


【五、添加背景音乐】

  把 .mp3 文件丢进这个文件夹即可：
  D:\ArkhamHorror\frontend\public\music\

  网页会自动读取该文件夹内的音乐，无需重启服务。


【六、启动的服务清单】

  服务                   端口      说明
  ─────────────────────────────────────────────────
  Docker 容器            3000      ArkhamHorror 游戏后端
  Vite Dev Server        8080      ArkhamHorror 前端开发服务器
  HTTPS 代理             3443      游戏主入口（代理到 8080）
  HTTPS 代理             3445      组牌器入口
  语音聊天服务           3001      WebSocket 语音聊天
  本地分享 API           -         组牌器分享功能


【七、关闭服务】

  方法一（推荐）：直接关机，下次开机再运行 start-all.ps1
  方法二（手动）：打开 PowerShell，运行：
    taskkill /IM node.exe /F
    docker compose -f D:\ArkhamHorror\docker-compose.yml down


【八、常见问题】

  Q: 浏览器显示"此连接不安全"？
  A: 因为是自签名证书，点"高级"→"继续前往"即可，这是正常的。

  Q: 页面打开但功能（音乐/语音/聊天室）没了？
  A: 重新运行 start-all.ps1，它会自动同步 tools 文件。

  Q: 手机/平板访问不了？
  A: 确保手机和电脑连的是同一个 WiFi，用电脑的 IP 地址访问。

  Q: Docker Desktop 没启动？
  A: start-all.ps1 会等待 Docker 就绪，但最多等 2 分钟。
     如果超时，请手动打开 Docker Desktop，再重新运行脚本。

  Q: 端口被占用？
  A: 运行以下命令关闭旧进程后重试：
     taskkill /IM node.exe /F

  Q: 组牌器分享链接（arkham.build）不工作？
  A: start-all.ps1 和 update-ip.ps1 都会自动修复 Vite 配置，
     重新运行脚本即可。
